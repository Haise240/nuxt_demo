# Портфолио-витрина + 3 демо-сайта

Единая система для GitHub Pages: одна **главная страница** (`index.html`) с общим текстом
об услуге и карточками-переходами на три живых демо. Всё собирается и публикуется **одним**
деплоем.

```
nuxt_demo/
├── index.html                 ← главная-витрина (лендинг)
├── .github/workflows/deploy.yml   ← единый деплой (собирает всё → GitHub Pages)
├── ossetia-cabins/            ← демо 1: горные дома (Nuxt 3)
├── ossetia-cafe/              ← демо 2: кафе осетинской кухни (Nuxt 3)
├── ossetia-tours/             ← демо 3: джип-туры (Nuxt 3)
└── plan-demo-sites.md         ← ваш рабочий план (НЕ публикуется на сайт)
```

После деплоя структура сайта:

```
/                     → витрина (index.html)
/ossetia-cabins/      → демо «Аланские горные дома»
/ossetia-cafe/        → демо «Фарн»
/ossetia-tours/       → демо «Дикий Кавказ»
```

Ссылки на карточках витрины — относительные, поэтому работают и под `user.github.io`,
и под проектным репозиторием без правок.

---

## Деплой на GitHub Pages

1. Создайте репозиторий на GitHub и запушьте в него **содержимое папки `nuxt_demo`** в ветку `main`.
2. В репозитории: **Settings → Pages → Build and deployment → Source: GitHub Actions**.
3. Любой push в `main` запускает workflow `deploy.yml`. Через пару минут сайт будет тут:
   - проектный репозиторий → `https://ВАШ_ЛОГИН.github.io/ИМЯ_РЕПО/`
   - либо назовите репозиторий `ВАШ_ЛОГИН.github.io` → сайт будет в корне `https://ВАШ_ЛОГИН.github.io/`

Базовые пути (`baseURL`) для демо вычисляются автоматически из имени репозитория —
ничего вручную прописывать не нужно.

## Локальный просмотр

Каждое демо — обычный Nuxt-проект:

```bash
cd ossetia-cafe        # или ossetia-cabins / ossetia-tours
npm install
npm run dev            # http://localhost:3000
```

Витрину `index.html` можно открыть прямо в браузере (карточки откроют демо, когда те собраны).
Чтобы локально проверить **всю** собранную систему, повторите шаги из `deploy.yml`:
соберите каждое демо (`npm run generate`), сложите `dist` в подпапки рядом с `index.html`
и поднимите любой статический сервер.

## Что заменить перед показом/боем

**`index.html`** (помечено комментариями `EDIT`):
- название/бренд в шапке и подвале (сейчас заглушка «Аланика Студия»);
- контакты в блоке «Связаться» (Telegram / WhatsApp / Email).

**В каждом демо:**
- фото — сейчас стоки-заглушки (в коде есть пометки); заменить на реальные кавказские/блюда;
- `YOUR_FORM_ID` в `composables/useApi.ts` → ваш Formspree или CRM-эндпоинт;
- телефоны, адрес и почту в шапке/футере/контактах.

## Заметки

- Файл `plan-demo-sites.md` **не попадает на сайт** (деплоятся только `index.html` и сборки демо),
  но **окажется в репозитории**. Если репозиторий публичный, а план показывать не хотите —
  удалите его или используйте приватный репозиторий.
- Внутри каждого демо лежит свой старый workflow `*/.github/workflows/nuxtjs.yml` — в этой
  единой системе он **не запускается** (GitHub Actions видит только workflow в корне репозитория).
  Можно оставить как есть или удалить.
